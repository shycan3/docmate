# 발표 보조 이미지 사용 가이드

생성일: 2026-05-21  
생성 스크립트: `scripts/generate_presentation_visuals.py`  
이미지 위치: `presentation-materials/visuals/`

이 문서는 발표 슬라이드에 바로 넣을 수 있도록 새로 만든 16:9 보조 이미지의 사용 목적을 정리한 자료입니다. `images/`와 `screenshots/`가 실제 서비스 화면을 보여준다면, `visuals/`는 심사위원이 서비스의 개념과 차별점을 빠르게 이해하도록 돕는 설명용 이미지입니다.

| 파일 | 역할 | 추천 위치 | 발표 포인트 |
| --- | --- | --- | --- |
| `01-service-positioning.png` | 서비스 한 장 소개 | 1. 오프닝 / 제품 소개 | DocMate가 단순 PDF 요약이 아니라 공고를 신청 준비 화면으로 바꾸는 서비스라는 점을 첫 장에서 명확히 보여준다. |
| `02-analysis-pipeline.png` | 분석 기술 흐름도 | 3. 기술 구조 / 구현 방식 | 현재 데모가 생성형 AI 없이 텍스트 추출, 규칙 기반 구조화, 프로필 비교, 근거 표시로 동작한다는 점을 설명한다. |
| `03-generative-ai-differentiation.png` | 생성형 AI PDF 업로드와의 비교 | 4. 차별화 / 심사 질문 대응 | "그냥 ChatGPT에 PDF 넣으면 되는 것 아닌가요?"라는 질문에 답할 때 사용한다. 핵심은 범용 요약이 아니라 신청 준비 워크플로우라는 점이다. |
| `04-trust-loop.png` | 신뢰성 설계 설명 | 5. 판정 근거 / 리스크 관리 | 결과를 확정 답변처럼 단정하지 않고, 원문 근거와 `추가 확인 필요` 상태를 통해 보수적으로 판정한다는 점을 강조한다. |
| `05-roadmap-readiness.png` | 발표 데모와 출시 준비도 구분 | 7. 한계와 발전 계획 | 경진대회 발표용 데모의 완성도와 실제 출시 전 필요한 검증 과제를 분리해서 말할 수 있다. |
| `06-privacy-local-processing.png` | 로컬 처리와 개인정보 메시지 | 6. 운영·보안 관점 | 현재 데모가 외부 AI API를 호출하지 않고 로컬에서 처리된다는 점을 설명한다. 출시 단계에서 추가되어야 할 보안 과제도 함께 말한다. |
| `07-summary-board.png` | 발표 말미 요약 보드 | 8. 결론 / Q&A 전환 | 문제, 해결, 차별점, 현재 강점, 남은 과제, 확장 방향을 한 화면에 정리해 발표를 안정적으로 마무리한다. |
| `08-judge-qna-map.png` | 심사위원 질문 대응 맵 | 9. 질의응답 준비 | 예상 질문을 가치, 기술, 신뢰성, 출시 가능성 네 갈래로 나누고 답변의 중심 문장을 고정한다. |

## 활용 전략

- 발표 초반에는 `01-service-positioning.png`와 실제 첫 화면 캡처를 이어 붙여 "무엇을 만드는 서비스인지"를 빠르게 잡는다.
- 기술 설명이 길어질 때는 `02-analysis-pipeline.png`만 보여주고 세부 구현은 질문이 들어왔을 때 답한다.
- 차별화 질문은 `03-generative-ai-differentiation.png` 한 장으로 대응한다. 이때 "ChatGPT보다 더 똑똑하다"가 아니라 "신청 준비 흐름을 표준화한다"라고 말한다.
- 한계와 로드맵은 `05-roadmap-readiness.png`로 정직하게 보여주는 편이 좋다. 발표용 완성도와 실제 출시 준비도를 분리하면 신뢰도가 올라간다.
- 마지막 장에는 `07-summary-board.png`를 띄워 두면 Q&A 중에도 서비스의 전체 논리가 화면에 남아 있다.

## 재생성 방법

디자인 문구나 색상을 바꾸고 싶으면 `scripts/generate_presentation_visuals.py`를 수정한 뒤 아래 명령을 실행한다.

```powershell
python scripts\generate_presentation_visuals.py
```

모든 이미지는 1600x900 PNG로 생성된다. 한글 폰트는 Windows 기본 `맑은 고딕`을 우선 사용한다.

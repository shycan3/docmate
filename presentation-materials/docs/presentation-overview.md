| 작품명 | DocMate: 장학금·청년정책 분석 콘솔 |

## 보고서 개요

DocMate는 장학금과 청년정책 공고를 읽는 데 드는 시간을 줄이고, 사용자가 실제 신청 준비까지 이어갈 수 있도록 돕는 정보시스템 프로젝트이다. 현재 많은 청년 지원 공고는 PDF나 긴 웹 공지 형태로 제공되어 신청 기간, 대상 조건, 소득 요건, 제출 서류, 중복지원 제한을 빠르게 파악하기 어렵다. 특히 구직자, 대학생, 저소득 청년처럼 여러 제도를 동시에 비교해야 하는 사용자는 자신에게 맞는 공고인지 판단하기 전에 상당한 탐색 비용을 부담한다.

본 프로젝트는 이 문제를 "공고 읽기"가 아니라 "신청 준비 의사결정"의 문제로 정의했다. 사용자는 나이, 학적, 거주지, 현재 상태, 소득구간, 재학 상태를 입력하고 PDF/TXT 공고 또는 샘플 공고를 선택한다. 시스템은 공고 텍스트에서 신청 기간, 지원 대상, 지원 내용, 준비 서류, 신청 방법, 신청 URL을 추출한 뒤 사용자 프로필과 비교해 신청 가능, 추가 확인 필요, 신청 불가 중 하나로 판정한다. 결과 화면은 결론을 먼저 보여주고, 이후 원문 근거와 위험 조건, 체크리스트를 단계적으로 확인하도록 구성했다.

구현은 경진대회 발표에 적합한 로컬 서비스 완성도를 목표로 했다. Python 표준 라이브러리 기반 API 서버와 SQLite 히스토리 저장소를 사용했고, 텍스트 기반 PDF 추출에는 PyMuPDF를 적용했다. 프론트엔드는 별도 빌드 없이 실행되는 HTML/CSS/JavaScript 구조로 만들었다. 기능적으로는 샘플 공고 3종 분석, 프로필 기반 판정, 근거 스니펫 표시, 위험 조건 탐지, 실행 체크리스트 생성, Markdown 내보내기, 히스토리 저장/삭제, 공고 비교, 발표용 데모 모드를 포함한다.

자료 조사와 개선 과정에서는 사용자 페르소나, 심사위원 페르소나, 미술·웹디자인 계열 페르소나의 피드백을 문서화하고 반영했다. 초기 화면이 번잡하다는 의견을 반영해 첫 화면을 `프로필 작성`과 `공고 선택` 중심으로 단순화했고, 거주지와 직업 같은 입력은 자유 입력 대신 선택형 컨트롤로 바꿨다. 또한 심사 관점에서 중요한 신뢰성 문제를 줄이기 위해 결과마다 원문 근거를 펼쳐 볼 수 있게 했고, 발표 중 기능 범위를 빠르게 보여주기 위해 데모 모드와 히스토리 비교를 강화했다.

시장 관점에서 DocMate의 주요 사용자는 장학금과 청년정책을 직접 찾아보는 대학생, 대학원생, 구직자, 미취업 청년이다. 확장 시에는 대학 행정팀, 청년센터, 지자체 정책 안내 조직이 공고 안내와 상담 효율화를 위해 사용할 수 있다. 단기적으로는 로컬 데모와 샘플 데이터 기반 발표 서비스이며, 중장기적으로는 실제 공고 수집, OCR, 사용자 계정, 개인정보 보호 정책, 캘린더 연동, 기관별 공고 추천으로 확장할 수 있다.

발표의 핵심은 세 가지이다. 첫째, 긴 공고를 읽기 전에 신청 가능성과 위험 조건을 먼저 확인하게 만든다. 둘째, 자동 판정의 근거를 원문 스니펫으로 확인하게 해 신뢰성을 확보한다. 셋째, 분석 결과를 준비 서류와 마감 확인 체크리스트로 바꿔 사용자의 다음 행동을 명확하게 만든다. 기대효과는 정보 탐색 시간 절감, 신청 누락 감소, 정책 접근성 향상, 공고 상담 업무의 반복 질문 감소이다.

## 팀원별 역할 제안

| 역할 | 발표에서 설명할 책임 |
| --- | --- |
| 기획/문제정의 | 청년 지원 공고 탐색 문제 정의, 사용자 흐름 설계, 경진대회 발표 메시지 정리 |
| 데이터/분석 | 공고 텍스트 추출 항목 정의, 규칙 기반 판정 로직, 위험 조건·체크리스트 구조 설계 |
| 백엔드 | 로컬 API, PDF/TXT 분석, SQLite 히스토리, 샘플 공고 API 구현 |
| 프론트엔드/UX | 프로필 입력, 결과 요약, 원문 근거, 히스토리 비교, 데모 모드 화면 구현 |
| 발표/검증 | 페르소나 피드백 정리, 심사 Q&A 준비, 발표자료와 시연 흐름 검증 |

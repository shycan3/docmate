# 발표 슬라이드 구성안

## 1. 제목

- 제목: `DocMate: 장학금·청년정책 분석 콘솔`
- 사용할 이미지: `images/01-first-screen-analysis-workspace.png`
- 메시지: 긴 공고를 사용자의 신청 준비 화면으로 바꾼다.

## 2. 문제 정의

- 청년 지원 공고는 길고, 조건이 흩어져 있으며, 제출 서류와 중복지원 제한을 놓치기 쉽다.
- 사용자는 "나에게 되는가"를 먼저 알고 싶지만 대부분의 공고는 문서 중심으로 제공된다.
- 메시지: 정보 부족보다 "판단과 행동으로 바꾸는 비용"이 문제다.

## 3. 사용자 흐름

- 사용할 이미지: `images/02-structured-profile-and-sample-choice.png`
- 프로필 작성 → 공고 선택 → 분석하기의 2단계 흐름을 설명한다.
- 선택형 입력을 통해 오타와 해석 차이를 줄인 점을 강조한다.

## 4. 핵심 결과 화면

- 사용할 이미지: `images/03-result-summary-decision.png`
- 신청 가능 여부, 기간, 위험 조건, 할 일을 먼저 보여준다.
- 메시지: 사용자는 긴 요약문보다 의사결정 요약을 먼저 필요로 한다.

## 5. 신뢰성: 원문 근거

- 사용할 이미지: `images/04-source-evidence-expanded.png`
- 추출 결과와 원문 근거가 연결된다는 점을 보여준다.
- 메시지: 자동화 결과를 맹신하게 하지 않고, 확인 가능한 구조로 만든다.

## 6. 행동 전환: 체크리스트

- 사용할 이미지: `images/05-checklist-and-next-actions.png`
- 제출 서류와 신청 방법이 체크리스트로 바뀌는 과정을 설명한다.
- 메시지: 분석 결과가 실제 신청 준비 행동으로 이어진다.

## 7. 히스토리와 비교

- 사용할 이미지: `images/06-history-comparison-demo-mode.png`
- 여러 공고를 저장하고 비교해 어떤 기회를 먼저 준비할지 판단할 수 있다.
- 메시지: 단일 문서 요약이 아니라 공고 탐색·비교 워크플로우다.

## 8. 구현과 확장

- 구현: Python API, PyMuPDF, SQLite, no-build frontend, 로컬 데모
- 검증: 사용자/심사위원/디자인 페르소나 피드백 반영
- 확장: 실공고 수집, OCR, 계정, 개인정보 보호, 캘린더 연동, 기관용 대시보드

## 9. 마무리

- 한 문장: DocMate는 청년 지원 공고를 "읽어야 하는 문서"에서 "바로 준비할 수 있는 실행 계획"으로 바꾼다.
- 시연 마무리: 데모 모드 실행 후 히스토리 비교 화면을 보여주고 Q&A로 연결한다.

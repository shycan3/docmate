# Initial Plan: docmate-mvp

## 요구사항 (무엇을 만들 것인가)
- 사용자가 정책 또는 장학금 공고 PDF를 업로드하면 신청 기간, 지원 대상, 혜택, 제출 서류, 신청 방법, 신청 URL을 구조화된 결과로 확인할 수 있게 만든다.
- 사용자가 나이, 학년, 거주지, 직업, 소득분위 같은 프로필을 입력하면 문서 조건과 비교해 신청 가능, 추가 확인 필요, 신청 불가 중 하나로 자격 결과를 표시한다.
- 문서의 제출 서류와 신청 절차를 즉시 실행 가능한 체크리스트로 변환하고 각 항목에 관련 이동 링크를 붙인다.
- 수정 불가, 중복 지원 불가, 자동 탈락, 국적 제한 같은 중요 조건과 제한 사항을 경고 카드로 강조한다.
- 자동 파싱된 신청 페이지 이동과 마감일 캘린더 등록을 제공하는 MVP 화면을 만든다.

## 구현 디테일 (어떻게 만들 것인가)
- 현재 가정은 React + TypeScript 프론트엔드와 FastAPI 백엔드로 구성하는 MVP이다.
- PDF 텍스트 추출은 Python 기반 라이브러리(PyMuPDF 우선, 필요 시 pdfplumber)를 사용한다.
- AI 분석은 구조화된 JSON 스키마를 기준으로 LLM 호출부를 분리하고, API 키가 없을 때도 개발과 데모가 가능하도록 샘플 분석 모드를 둔다.
- 자격 판정은 MVP 범위에서 단순 조건은 rule-based로 비교하고, 복합 조건이나 누락 정보는 보수적으로 "추가 확인 필요"로 처리한다.
- 첫 구현은 1~5페이지 텍스트 기반 정책·장학금 PDF와 대학생·청년 대상 공고에 집중한다.

## 제약 조건 (무엇을 안 할 것인가)
- 초기 MVP에서는 로그인, 결제, 구독, 기관용 관리자 기능을 구현하지 않는다.
- 초기 MVP에서는 이미지 위주 PDF, 복잡한 법률 문서, 표가 대부분인 문서, 다단계 예외 규칙을 완전 자동 판정하지 않는다.
- 초기 MVP에서는 실제 Google OAuth 기반 캘린더 쓰기 연동 대신 캘린더 등록 링크 또는 ICS 다운로드 수준으로 제한한다.
- 초기 MVP에서는 정부24, 학교 포털, 지자체 사이트별 자동 로그인이나 서류 자동 발급을 구현하지 않는다.

## 완료 기준 (이번 라운드의 끝)
- `python -m pytest` 또는 프로젝트에 맞는 테스트 커맨드가 통과한다.
- 개발 서버 실행 후 브라우저에서 PDF 업로드, 프로필 입력, 구조화 결과, 자격 상태, 경고 카드, 체크리스트, 신청 링크, 마감일 등록 동작을 확인할 수 있다.
- 샘플 공고 문서 또는 샘플 텍스트로 API 키 없이도 데모 흐름을 끝까지 확인할 수 있다.
